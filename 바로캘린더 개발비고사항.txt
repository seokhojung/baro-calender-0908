## 📝 **커밋 로그 스타일 이슈 정리**

### 🐛 **Fix: Docker 환경 충돌 및 포트 설정 문제 해결**

#### **문제 상황:**
- PC 환경 변경 후 도커 실행 시 네트워크 IP 충돌 발생
- 포트 매핑 불일치로 프론트엔드 접속 불가
- 기존 프로젝트 잔여물로 인한 컨테이너 이름 중복

#### **에러 메시지:**
```
Error: Pool overlaps with other one on this address space
Error: Container name is already in use
Error: listen EADDRINUSE: address already in use :::3000
```

#### **해결 내용:**
```bash
# 도커 환경 정리
docker system prune -f  # 12.37GB 캐시 정리

# 포트 매핑 수정
- ports: "3000:3000" → "8000:8000"  # 백엔드를 올바른 포트로

# 네트워크 설정 변경
- subnet: 172.21.0.0/16 → 172.22.0.0/16  # IP 충돌 해결
```

#### **결과:**
- ✅ 백엔드: `http://localhost:8000` (정상 작동)
- ✅ 프론트엔드: `http://localhost:3000` (정상 작동)

#### **추가된 섹션:**
- `⚠️ PC 변경 시 필수 체크리스트` - 도커 환경 정리 가이드
- `PC 변경 후 자주 발생하는 문제들` - 실제 경험 기반 트러블슈팅
